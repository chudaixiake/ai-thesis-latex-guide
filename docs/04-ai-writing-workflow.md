# 用 Codex 或 Claude Code 写论文

AI 辅助写论文的关键不是一次生成全文，而是把论文当成一个可版本管理的工程。

## 推荐流程

1. 建立论文项目目录。
2. 写 `thesis-config.md`，记录题目、专业、研究问题、方法、数据和学校格式要求。
3. 写 `outline.md`，先确定章、节、小节。
4. 建 Zotero 文献分类并自动导出 `.bib`。
5. 按章节逐段写作。
6. 每次改完一章就编译。
7. 统一检查引用、图表、公式、格式和致谢。

## 适合给 AI 的任务

```text
根据 thesis-config.md，帮我细化第 1 章大纲。
```

```text
基于 references.bib 中已有文献，帮我写第 2.1 节文献综述，不要编造引用。
```

```text
检查 main.tex 和 chapters 目录，找出 LaTeX 编译风险。
```

```text
把这一节改得更像中文学位论文，但不要增加没有来源的结论。
```

## 不适合直接交给 AI 的任务

- 编造实验结果
- 编造参考文献
- 替你确认学校最新格式要求
- 隐去开源模板来源或规避许可证

## 建议的项目规则

在论文项目里放一个 `AGENTS.md`：

```md
- 不编造文献、数据、实验结果、页码和 DOI。
- 引用必须来自 references.bib 或用户提供的材料。
- 正文语言使用正式中文。
- 修改 LaTeX 后尽量运行 latexmk -xelatex main.tex。
- 保持学校模板格式，不做无关样式改造。
```

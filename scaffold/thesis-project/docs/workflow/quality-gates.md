# 质量门禁

本文件给 AI 和作者提供每轮写作后的检查清单。能用命令检查的优先用命令；不能自动检查的，必须在工作日志中说明人工判断依据。

## 写作真实性

- 不编造参考文献、DOI、页码、作者、期刊、数据、实验结果或访谈材料。
- 重要观点应有 citation key、数据来源、实验记录或用户提供材料支持。
- 无法确认的内容保留明显标记：`[需要引用]`、`[需要数据]`、`[待确认]`。

## 引用与文献

- 新增 `\cite{...}` 前，优先确认 key 存在于 `docs/references.bib`。
- 新读文献应写入 `docs/notes/{citation-key}.md`，并更新 `docs/literature-matrix.md`。
- 不手写虚构 BibTeX；文献库以 Zotero Better BibTeX 导出的 `docs/references.bib` 为准。

## LaTeX 与模板

- 正式编译路径保持为：

```powershell
cd paper
latexmk -xelatex main.tex
```

- 模板文件、字体、页边距、页眉页脚、标题、目录、参考文献样式属于格式关键部分，未经用户确认不要修改。
- 使用第三方模板前先检查许可证和署名要求。
- 学校或期刊格式要求以目标单位最新官方文件为准，本项目说明不是官方规范。

## 文件范围

- 正文写作优先修改 `paper/chapters/`。
- 项目信息优先修改 `docs/thesis-config.md`。
- 大纲优先修改 `docs/outline.md`。
- 文献笔记写入 `docs/notes/`，不要散落在聊天记录里。
- 导师 Word 批注先整理到 `docs/worklog/` 或修订路线图，再回填 LaTeX。

## 每轮交付前

至少检查：

- 本轮是否有工作日志或修改摘要。
- 是否说明输入材料和修改范围。
- 是否留下未解决问题清单。
- 是否运行过可用的检查命令；若未运行，说明原因。
- 是否误改格式关键文件。

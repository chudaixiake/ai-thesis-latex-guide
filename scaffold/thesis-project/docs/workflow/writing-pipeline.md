# AI 写作流程

本流程用于约束 Codex / Claude Code 的论文写作节奏，避免直接跳到整篇生成。适用于学位论文、期刊论文、会议论文或其他 LaTeX 文档。

## 总原则

- 先确认目标、资料和格式来源，再写正文。
- 重大改动先写计划，再修改文件。
- 每次只推进一个明确任务，如一章、一节、一次模板适配或一轮导师批注。
- 所有重要结论必须能追溯到文献、数据、实验结果或用户提供材料。
- 没有来源的位置用 `[需要引用]`、`[需要数据]` 或 `[待确认]` 标记。

## 推荐阶段

| 阶段 | 输入 | 产出 | 通过条件 |
| --- | --- | --- | --- |
| 1. 项目配置 | `docs/thesis-config.md` | 明确题目、对象、方法、目标格式 | 无关键字段空缺，或已标记待确认 |
| 2. 大纲确认 | `docs/outline.md` | 章节结构和每章写作目标 | 用户确认或明确要求继续 |
| 3. 文献整理 | `docs/references.bib`、PDF、笔记 | `docs/notes/`、`docs/literature-matrix.md` | citation key 可追溯，不编造文献 |
| 4. 写作计划 | 本次任务说明 | `docs/worklog/{date}-{task}/plan.md` | 范围、输入、输出、验收标准清楚 |
| 5. 单元写作 | 大纲、笔记、参考文献 | `paper/chapters/*.tex` | 只改目标章节，保留待补来源标记 |
| 6. 引用检查 | 章节文件、`docs/references.bib` | 引用问题清单或修复 | `\cite{}` 尽量能在 BibTeX 中找到 |
| 7. 学术润色 | 已写章节 | 修改后的章节 | 不改变事实、数据和引用含义 |
| 8. LaTeX 验证 | `paper/main.tex` | 编译结果或问题记录 | 优先运行 `latexmk -xelatex main.tex` |
| 9. Word 审阅 | `docs/export/`、`docs/review/` | 修改路线图和回填修改 | 批注先归纳，再改回 LaTeX |
| 10. 交付总结 | 本轮修改记录 | `summary.md` | 说明改了什么、未完成什么、如何验证 |

## 人工确认点

以下情况应先向用户确认：

- 更换学校或期刊 LaTeX 模板。
- 修改字体、页边距、页眉页脚、标题、目录、参考文献样式。
- 删除大段正文、章节或参考文献。
- 根据不完整资料补写实验结果、数据分析或结论。
- 将 Word 批注解释为实质性研究结论变化。

## 失败回退

- 编译失败：先定位 LaTeX 语法、路径、引用或模板问题，不继续扩写正文。
- 引用缺失：保留 `[需要引用]`，不要临时编造 BibTeX。
- 格式要求不明：标记 `[待确认格式要求]`，以学校或期刊最新官方文件为准。
- 文献证据不足：降低表述强度，或把段落移入待补材料清单。
